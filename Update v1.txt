4. Priority days for ASAP shipping. Repeater field inside the asap column, you add a custom "send out date" and its priority over another priority day. Essentially it doesn't change the logic of conditional application to asap shipping, it changes when the order would be sent out.
For example currently a user has macarons and dubajaus šokoladas in his cart. Venipak shipping method is set as asap, sending days are Thursday and max ship time is 2 work days. So today is 2026-01-11, the user seets the message "Pristatymas ne vėliau kaip pirmadienis, 2026-01-19". With the new feature the admin can separately add priority sending days which take effect over the normal sending days. For example the admin adds priority day:
date=2026-02-16, label=Kalėdinė rezervacija, priority=1. He then drags dubajaus šokoladas from the normal column to the priority day column. Now a user has macarons and dubajaus šokoladas in his cart. Venipak shipping method is selected and the date 2026-01-11 but he sees the message "Pristatymas ne vėliau kaip pirmadienis, 2026-02-22" because he has that priority item which effects. This update does not require all the categories to be in both non priority days and priority days, the method is available no matter where its added, rather what changes is the delivery date.
